================================================================================
                    Building a Todo App - Python & JS Complete Guide
================================================================================


================================================================================
Overview
================================================================================

This course teaches how to build a complete Todo application from scratch
using Flask (Python) and vanilla JavaScript. Students will learn Flask basics,
database setup, user authentication, CRUD operations, protected routes,
and admin functionality step-by-step.

Target Audience: Beginners who want to learn full-stack web development
Prerequisites: Basic Python, Basic JavaScript, HTML/CSS fundamentals
Total Parts: 8 modules building a complete Todo application


================================================================================
Features
================================================================================

- 8 Progressive modules building complexity step-by-step
- Each module focuses on ONE concept
- Flask backend with SQLAlchemy database
- Vanilla JavaScript frontend with Bootstrap 5
- JWT token-based authentication
- Role-based access control (Admin/User)
- Password hashing with Werkzeug
- Detailed comments explaining every concept
- Each part has complete working code
- Helper functions used instead of decorators (for easier learning)

================================================================================
                         Part-1: Hello Flask
================================================================================
================================================================================

Overview
--------
Learn the foundation of Flask web development. Understand how to create a
Flask application, define routes, and render HTML templates. This is the
starting point for building any Flask web application.

Features
--------
- Creating a Flask application
- Defining routes (URLs)
- Rendering HTML templates
- Running the development server
- Understanding debug mode

Instructions
------------
Navigate to the part-1-hello-flask folder in your terminal.

Create virtual environment: python -m venv venv

Activate it: venv\Scripts\activate (Windows) or source venv/bin/activate (Mac/Linux)

Install dependencies: pip install -r requirements.txt

Run the server: python app.py

Open http://127.0.0.1:5000 in your browser.

Key Learnings
-------------
Flask App: Creating application with Flask(__name__).

Routes: Using @app.route('/') to map URLs to functions.

render_template(): Rendering HTML files from templates folder.

debug=True: Auto-reload server when code changes.

URL Mapping: Each route connects a URL to a Python function.

Activity Instructions
---------------------
Open part-1-hello-flask folder and complete these exercises.

Activity 1: Add a new route @app.route('/services') and create services.html template.

Activity 2: Change the port to 8080 using app.run(debug=True, port=8080).

Activity 3: Create a route that returns plain text instead of HTML template.

Activity 4: Create dynamic route @app.route('/hello/<name>') that greets by name.

                         Part-2: Database Setup
================================================================================
================================================================================

Overview
--------
Add persistent data storage using SQLite database with SQLAlchemy ORM.
Learn how to create database models (tables), connect Flask to database,
and perform basic database queries.

Features
--------
- SQLAlchemy ORM integration
- User and Todo models (tables)
- Database initialization
- Basic database queries
- Test route to verify database

Instructions
------------
Navigate to the part-2-database-setup folder.

Create and activate virtual environment.

Install dependencies: pip install -r requirements.txt

Run the server: python app.py

Visit http://127.0.0.1:5000/test-db to test database.

Key Learnings
-------------
SQLAlchemy: ORM (Object Relational Mapper) to work with databases using Python.

Database URI: 'sqlite:///todo.db' creates SQLite database file.

Models: Python classes that represent database tables.

db.Column(): Defining table columns with types (String, Integer, Boolean).

db.relationship(): Creating relationships between tables (User has many Todos).

db.session.add(): Adding new record to database session.

db.session.commit(): Saving changes to database.

Query Methods: .query.all(), .query.first(), .query.filter_by().

Activity Instructions
---------------------

Activity 3: Install "DB Browser for SQLite" and view the todo.db file.

Activity 4: Modify test_db() to create 3 users with different todos.

                         Part-4: User Login
================================================================================
================================================================================

Overview
--------
Implement secure user login with password hashing and JWT tokens.
Learn how to hash passwords for secure storage, verify passwords on login,
and generate JWT tokens for authentication.

Features
--------
- Secure password hashing
- Password verification
- JWT token generation
- Login API endpoint
- Dashboard page after login
- Token stored in localStorage

Instructions
------------
Navigate to the part-4-user-login folder.

Set up virtual environment and install dependencies.

Run the server: python app.py

Register a new user first, then login with same credentials.

Check browser DevTools > Application > Local Storage for stored token.

Key Learnings
-------------
Password Hashing: Using hash_password() to convert plain text to hash.

Password Verification: Using verify_password() to check password against hash.

JWT Token: JSON Web Token containing user_id and is_admin.

create_token(): Generate JWT token on successful login.

localStorage: Browser storage for keeping token on client side.

401 Unauthorized: Response when credentials are invalid.

Login Flow: Receive credentials -> Verify -> Generate token -> Return to client.

Activity Instructions
---------------------
Open part-4-user-login folder and complete these exercises.

Activity 1: Try logging in with wrong password and observe the error.

Activity 2: Check the token in localStorage and decode it at jwt.io.

Activity 3: Add 'remember_me' parameter - 30 day expiration if true, else 1 hour.

Activity 4: Add login attempt logging to console.

                         Part-6: Protected Routes
================================================================================
================================================================================

Overview
--------
Secure the API endpoints so only authenticated users can access their own data.
Learn how to validate JWT tokens and extract user information for protected
routes. Uses helper function instead of decorator for easier understanding.

Features
--------
- Token validation on every request
- get_current_user() helper function
- User can only access their own todos
- Ownership verification
- 401/403 error responses

Instructions
------------
Navigate to the part-6-protected-routes folder.

Set up virtual environment and install dependencies.

Run the server: python app.py

Login to get a token, then access todo endpoints.

Try accessing without token to see 401 error.

Key Learnings
-------------
Authorization Header: Sending token as "Bearer <token>" in request header.

get_current_user(): Helper function that validates token and returns user.

Token Extraction: Parsing token from "Bearer <token>" format.

Token Validation: Decoding JWT and checking if valid/expired.

Ownership Check: Verifying todo belongs to current user before update/delete.

401 Unauthorized: No token or invalid token.

403 Forbidden: Valid token but not authorized for this resource.

Activity Instructions
---------------------
Open part-6-protected-routes folder and complete these exercises.

Activity 1: Try accessing /api/todos without Authorization header.

Activity 2: Try to update/delete another user's todo and observe 403 error.

Activity 3: Add a /api/profile endpoint that returns current user's info.

Activity 4: Add logging to see which user is making each request.

================================================================================
                         Key Concepts Summary
================================================================================

Flask Basics
------------
- Flask(__name__) creates the application
- @app.route() maps URLs to functions
- render_template() renders HTML files
- request.get_json() parses JSON from request body
- jsonify() converts dict to JSON response

Database (SQLAlchemy)
---------------------
- db.Model creates a table
- db.Column() defines columns
- db.session.add() adds records
- db.session.commit() saves changes
- Model.query.filter_by() searches records

Authentication
--------------
- hash_password() securely stores passwords
- verify_password() checks password on login
- create_token() generates JWT token
- get_current_user() validates token and returns user
- Authorization header: "Bearer <token>"

HTTP Status Codes
-----------------
- 200: OK (success)
- 201: Created (new resource created)
- 400: Bad Request (invalid input)
- 401: Unauthorized (no token or invalid token)
- 403: Forbidden (not allowed to access resource)
- 404: Not Found (resource doesn't exist)
